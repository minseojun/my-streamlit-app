## ✅ 수행한 심화 항목: A / B

## 연동한 API / 추가한 기능:
1) 외부 API 연동(A)
Open-Meteo API: 도시 검색(geocoding) + 날짜별 날씨(archive/forecast) 조회 → Planner 화면에서 선택 날짜의 날씨 표시

2) UX/기능 고도화(B)
a. 사용자별 개인화(로그인 대체): URL ?uid= 기반 고정 user_id로 새로고침/재접속에도 동일 사용자 데이터 유지
b. 입력 기록 저장/히스토리: 계획/습관/성공·실패·미완료 상태/실패 원인 등을 SQLite에 영구 저장 → 날짜/주 단위로 히스토리 조회 가능
c. 결과 내보내기: 주간 기록을 PDF 리포트로 생성/다운로드(한글 폰트 임베딩 대응)
d. 데이터 시각화 대시보드:
  이번 주 요일별 실패 분포 차트
  최근 N주(기본 8주) 실패 원인 카테고리 트렌드 라인 차트(TOP 6, 실제 횟수 기준)
e. 리마인더 UX: 설정한 시간대에 미완료 항목이 있으면 toast 알림
f. 설정 저장 UX: 날씨 도시/표시 여부, 리마인더 설정, OpenAI 키/모델 등을 쿠키 기반 저장(가능한 환경에서)
g. 민감정보/AI 동의 UI: 앱 하단에 데이터 저장/AI 사용 안내 + 동의 체크 추가(동의 없으면 AI 기능 비활성)
(추가 예정/반영 가능) 개인 데이터 삭제 기능: 사용자 데이터 일괄 삭제(하단에 안전장치 포함)



## 심화를 통해 개선된 점:

단순 체크리스트를 넘어, 실패 원인을 데이터로 축적하고 패턴(요일/기간/카테고리 트렌드)을 시각적으로 파악할 수 있게 됨
사용자가 입력한 다양한 실패 원인을 그대로 나열하지 않고, AI가 의미 기반으로 묶어주는 카테고리화로 인사이트가 훨씬 직관적이 됨
PDF 리포트로 결과를 외부 공유/보관할 수 있어 “기록 → 분석 → 회고” 흐름이 완성됨
로그인 없이도 고정 user_id로 사용자별 데이터가 안정적으로 유지되어 실사용성이 크게 개선됨
민감 데이터에 대한 안내/동의 단계를 추가해 신뢰성과 제품 완성도가 올라감
